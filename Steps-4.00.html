Problems Encountered 4

1. Data transformation and manipulation:  The latest dashboard you can view under “Executive Summary” above uses four datasets which means quit a bit of transformation and manipulation to go from the raw data to the final product.  Once specific example of a transformation was the DUAL () function.  The raw data simply had the date in this format, 2016-06-14T04:29:57Z .  Taking the raw date data, concatenating the four data sets, then using the DUAL () function properly, the end product is what you see as the dates in order by year and month on the X-axis of the Sales By Year-Month table.  It took awhile for me to Understand this function and get the script correct with lots and lots of trials and tribulations but in the end it was totally worth it because this is another tool in the toolbox that is now part of my vernacular.    

2.  Dataset:  Problem Number One: Depth of the original dataset and data model. When I created this data model from the original dataset there was one thing missing, time. I went back and enhanced the data set with more in depth data to create a more complete dataset and data model. Time is not an issue anymore. The data set grew from one table to multiple tables to truly be a set of data enabling to paint a much better and more in depth picture with time now being part a dimension. This new dataset should allow me to really dig into the dataset. 

3. KPIs:  What KPIs to use, how to get all the information into a dashboard where it makes sense and not overwhelming.  The below pictures are part of a process that took quite a bit of time and energy to figure out the layout (with guidance from Ali), the flow, the placement, of each KPI, and the color scheme.  As you can see from the screenshots, the KPIs start as a discombobulated mess and end professionally designed.

4. The Map:  The map was a major process from not really understanding how to build a functional map at first to finishing with multiple layers that show the different regions as an Area Map with a Point Map on top of that layer to give an in depth representation of dataset by location.  Without a doubt, to go from the first map to the last took a few months BUT every step was a step of learning something new and taking a step forward.  The two most challenging parts of the final map was one, making a mapping table using Mapping Load, then connecting it to the Fact Table using the ApplyMap() function along with the different regional colors; two, I really spent a lot of time trying to move Alaska and Hawaii from their Qlik Sense designated positions to below the continental U.S., and shrinking AK at the same time.  I actually did that using KML files but I couldn’t get the AK/HI data to be recognized as new states by the point layer.  AK/HI’s point layers kept being recognized in their normal positions and not through the KML.  That was a total blast learning everything about KML files and how to manipulate them.
