 Transforming Data in the Main section

Below, the four data sets need to be transformed and CONCACTENATED into one FACT TABLE.  This is the scripting of the
four datasets with transformations.  There are 3 concatenations so to help the four datasets become one fact table.  "Invoice Date "
was kept and transformed into the four different fields of, 'InvoiceDate', 'InvoiceDay', 'InvoiceMonth', and 'InvoiceYear' to extract
the different parts of the date.  The DUAL statement is what got the months and dates extracted and in order as they look in the
"Sales by Year-Month" combo chart that you will make in future steps.  The WHERE clause really helped pull dates only from
the year 2020 forward.  Since the datasets came dirty, the where clause helps clean up the data by only pulling the dates/years
from 2020 forward.  There was not a lot of pre-2020 dates but enough to clean them up. 


The 4th concatenation below is a Resident load that adds two fields: `GroupedRegion`, which categorizes states into regions
like 'Northeast' or 'West', and `RegionColor`, which assigns colors like red or green to these regions based on the state. It uses
nested `If` and `Match` functions to determine the region and color code by matching states to predefined lists. The `Resident`
keyword indicates that the data is loaded from an already existing table in memory. This enhancement facilitates regional
analysis and visualization by adding meaningful labels and color coding to the data set.
The 5th concatenation also uses a Resident load that adds a `Season` field  by categorizing the `InvoiceDate` into 'Winter',
'Spring', 'Summer', or 'Fall', based on the month. This is achieved by using conditional `If` statements within a `RESIDENT`
data load.  The 6th concatenation transforms the `ShoeFactTable` by adding `MonthOrder2020` and `MonthOrder2021`,
fields that extract the invoice month for the years 2020 and 2021 and assigns `null` to other years, thereby organizing the
data annually.


The 7th concatenation transforms the data into a format for all "Invoice Date" entries into a consistent manner and creates
a new column that uniquely identifies each invoice by its month and distinguishes between those from the year 2020 and all
others. This enables easier sorting and comparison of invoices by month, particularly between the years 2020 and 2021 in the
ShoeFactTable.  The 8th concatenation transforms the data by adding a new column `MonthName` that converts the numeric
month from the `InvoiceDate` field into its corresponding month name, like 'January' for 1, 'February' for 2, etc. Using nested
`If` statements that check the month number and assign the appropriate name, simplifying month identification for analysis
or reporting. The 9th concatenation combines the year and month from each "Invoice Date" into a new column
`YYearMMonth`, displayed as "2020-01" for January 2020, using `Dual` to keep it both readable and sortable.
